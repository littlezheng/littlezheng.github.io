    分支管理策略
  • master分支非常穩定，僅用於發佈新版本
  • dev分支不穩定，平時工作都是從其他分支合併到dev分支
  • bug分支，處理Bug的分支
  • feature分支，創建新特徵的分支
    多人協作策略
  • 試圖git push origin your_branch_name推送自己的修改
  • 若推送失敗，則因爲遠程分支比你的本地分支更新，需要試圖git pull合併
  • 若git pull提示"no tracking information"，則本地分支未與遠程分支創建鏈接關係，試圖git branch --set-upstream your_branch_name origin/branch_name
  • 若合併衝突，則解決衝突，並在本地git commit
  • 若無衝突，或衝突已解決，則試圖git push origin your_branch_name
    Git基本操作
  • 創建空的倉庫：git init（git init --bare，用於Git服務器端創建空的倉庫）
  • 克隆遠程倉庫：git clone https://xxx.xxx/xxx.git
  • 添加文件到緩存區：git add .（git add your_filename）
  • 提交緩存到倉庫：git commit -m "your_commment"
  • 創建tag：git tag -a your_tag -m "your_commment"（git tag -d your_tag，刪除tag）
    Git查看狀態及差異
  • 查看當前倉庫狀態：git status（git status -s 簡單方式查看）
  • 查看文件的差異：git diff your_filename（git diff HEAD -- your_filename）
  • 查看提交日誌：git log（git log --pretty=oneline，簡單方式顯示）
  • 當前版本爲HEAD，上一個版本爲HEAD^，上上個版本爲HEAD^^，上100個版本爲HEAD~100
  • 回退到以前的版本：git reset --hard HEAD^（git reset --hard your_commit_id）
  • 查看所有提交命令：git reflog
    Git分支
  • 查看所有分支：git branch
  • 創建或切換分支：git checkout -b your_branch_name
  • 將其他分支合併到當前分支：git merge your_branch_name（git merge your_branch_name --no-ff -m "your_commment"，禁用fast forward快速合併，強制一個新的提交）
  • 刪除分支：git branch -d your_branch_name
  • 分支衝突的話，必須先解決衝突，再提交
  • 查看分支合併圖：git log --graph
    Git遠程倉庫
  • 查看遠程倉庫，默認爲origin：git remote（git remote -v 詳細信息）
  • 添加本地分支與遠程分支的關聯：git remote add origin your_branch_name
  • 從遠程分支拉到本地：git pull（若git pull提示"no tracking information"，則本地分支未與遠程分支創建鏈接關係，試圖git branch --set-upstream your_branch_name origin/branch_name）
  • 推送本地分支到遠程倉庫：git push origin your_branch_name
